= Hardware Support
:description: List of processor architectures and toolchains supported by the ThreadX source tree.

This page lists the processor architectures for which a port is maintained in the ThreadX source tree, together with the toolchains each port currently supports. The list is derived directly from the contents of the `ports/` and `ports_smp/` directories of the ThreadX repository, and is therefore authoritative for the current release.

Ports are grouped by architecture family. Single-core ports are covered first, followed by the SMP ports. Simulation ports, used for development and testing on a host PC, are listed at the end.

== ARM Cortex-M (single-core)

|===
| Core | Toolchains

| Cortex-M0 / M0+ | AC5, AC6, GNU, IAR, Keil
| Cortex-M3       | AC5, AC6, GHS, GNU, IAR, Keil
| Cortex-M4       | AC5, AC6, GHS, GNU, IAR, Keil
| Cortex-M7       | AC5, AC6, GHS, GNU, IAR
| Cortex-M23      | AC6, GNU, IAR
| Cortex-M33      | AC6, GNU, IAR
| Cortex-M55      | AC6, GNU, IAR
| Cortex-M85      | AC6, GNU, IAR
|===

The Cortex-M23, M33, M55 and M85 ports implement the ARMv8-M architecture, including TrustZone for ARMv8-M.

== Notes

* The information above reflects the state of the ThreadX repository at the time of writing. The authoritative source is always the set of directories under `ports/` and `ports_smp/` in the ThreadX source tree.
* Toolchain abbreviations: *AC5* and *AC6* refer to ARM Compiler 5 and 6, *GHS* to Green Hills Software, *GNU* to the GNU toolchain (GCC/Binutils), *IAR* to the IAR C/C++ Compiler, *Keil* to the Keil MDK toolchain, *CCS* to Texas Instruments Code Composer Studio, *CC-RX* to the Renesas CC-RX compiler, *MetaWare* to the Synopsys MetaWare toolchain, *XCC* to the Cadence Xtensa C/C++ Compiler, and *Clang* to the LLVM/Clang toolchain.
* Historical ports for architectures no longer present in this repository (for example Analog Devices SHARC/Blackfin, Infineon TriCore, Microchip PIC/SAM, Renesas SH/V850, Intel x86/NIOS II, Xilinx MicroBlaze/PowerPC) may be available in archived branches or from the original Express Logic distribution, but are not maintained as part of the current Eclipse ThreadX source tree.
